🚗📈 MLOps Pipeline de Previsão de Vendas Chevrolet 2024

Documentação Completa da Arquitetura e Fluxo de Dados

✨ Introdução ao Pipeline MLOps

Este documento detalha um pipeline MLOps completo, projetado para gerenciar o ciclo de vida de um modelo de Machine Learning focado na previsão de vendas de veículos Chevrolet para o ano de 2024. Desde a geração e ingestão de dados até o treinamento, registro, serviço e teste do modelo, e a geração de relatórios analíticos, cada etapa é orquestrada para garantir eficiência, rastreabilidade e confiabilidade.

O objetivo é demonstrar uma arquitetura de dados e ML robusta, utilizando ferramentas como Flask para APIs, Prefect para orquestração de dados, e MLflow para gerenciamento de experimentos e modelos. O foco está no fluxo de informações e na interação entre os diferentes serviços.

🛠️ Componentes do Pipeline

Cada arquivo Python representa um serviço ou componente chave com uma responsabilidade específica:

🏭

API API-DATA-SERVER-V1.PY

Função: Simula a fonte de dados original, gerando dados de vendas fictícios, porém realistas, para concessionárias Chevrolet em 2024.

Interage com: Ninguém (fonte inicial de dados).

Disponível em: Porta 8777 - Endpoint /sales_data

📥

Prefect Flow PREFECT-DATA-INJECTION-FLOW-V1.PY

Função: Orquestra a ingestão de dados. Busca dados da API de vendas (8777) e os envia para a API de Storage (8778).

Interage com: API-DATA-SERVER-V1.PY (GET) e API-OBJECT-STORAGE-SERVER-V1.PY (POST).

Ferramenta: Prefect para orquestração de fluxos de trabalho de dados.

🗄️

API API-OBJECT-STORAGE-SERVER-V1.PY

Função: Atua como um Data Lake passivo, recebendo e armazenando dados brutos de vendas em um banco de dados SQLite (sales_data_lake.db).

Interage com: Recebe dados via POST do Prefect Flow (ingestão) e serve dados via GET para os pipelines MLflow (treinamento e análise).

Disponível em: Porta 8778 - Endpoints /store_data (POST) e /datasource (GET)

📊

MLflow Server ML-FLOW-SERVER-V1.PY

Função: Inicia o MLflow Tracking Server, essencial para rastrear experimentos de ML, logar métricas, parâmetros, artefatos e gerenciar o Model Registry.

Interage com: Todos os pipelines MLflow para rastreamento e registro de modelos.

Disponível em: Porta 5001 - UI do MLflow

🧠

MLflow Pipeline ML-FLOW-DATA-PIPELINE-MODEL-TRAINER-GRAPH-REPORT-EXPORTER.PY

Função: O coração do treinamento ML. Busca dados do Data Lake (8778), pré-processa-os, treina um modelo Keras para previsão de vendas, avalia-o, registra o modelo (incluindo pré-processador) no MLflow Model Registry e gera relatórios e gráficos de desempenho. Também faz upload de artefatos para o Artifact Storage (8779).

Interage com: API-OBJECT-STORAGE-SERVER-V1.PY (GET), MLflow Server (log, registro), API-SERVER-DATA-STORAGE-OUTPUT-V1.PY (upload).

Modelagem: Usa TensorFlow/Keras, scikit-learn.

📦

API API-SERVER-DATA-STORAGE-OUTPUT-V1.PY

Função: Um serviço de armazenamento de artefatos. Recebe arquivos (relatórios, gráficos) via POST e os armazena em um diretório local. Permite acesso via GET para download.

Interage com: Pipelines MLflow (trainer e gerador de gráficos) para upload de artefatos.

Disponível em: Porta 8779 - Endpoints /upload_artifact (POST), /artifacts/<filename> (GET)

🚀

API API-SERVER-MODEL-LOAD-FROM-ML-FLOW--PREDICT-SERVICE-V1.PY

Função: O serviço de inferência do modelo. Carrega a versão mais recente (ou especificada) do modelo ChevroletSalesPredictor do MLflow Model Registry e expõe um endpoint POST para receber novas requisições e retornar previsões.

Interage com: MLflow Server (para carregar o modelo) e API-SERVER-MODEL-TESTER-V1.PY (recebe requisições de teste).

Disponível em: Porta 8780 - Endpoints /predict (POST) e /model_status (GET)

🧪

Tester API-SERVER-MODEL-TESTER-V1.PY

Função: Um cliente de teste para o modelo em produção. Gera dados sintéticos, os envia para a API de serviço do modelo (8780), recebe previsões e gera um relatório de teste detalhado com gráficos.

Interage com: API-SERVER-MODEL-LOAD-FROM-ML-FLOW--PREDICT-SERVICE-V1.PY (GET status, POST predict).

📈

MLflow Pipeline ML-FLOW-DATA-PIPELINE-GRAPH-GEN-V1.PY

Função: Um pipeline de análise de dados e geração de relatórios/gráficos. Busca dados brutos do Data Lake (8778), processa-os (ex: encontra os top 10 modelos mais vendidos), gera um gráfico e o loga no MLflow e também faz upload para o Artifact Storage (8779).

Interage com: API-OBJECT-STORAGE-SERVER-V1.PY (GET), MLflow Server (log de artefatos), API-SERVER-DATA-STORAGE-OUTPUT-V1.PY (upload).

🌐

Prefect Server PREFECT-SERVER-V1.PY

Função: Inicia o servidor local do Prefect, fornecendo a UI e o backend para monitorar e orquestrar os flows Prefect (como o de ingestão de dados).

Interage com: O Prefect Flow de ingestão.

Disponível em: Porta 4200 - UI do Prefect

🌊 Fluxo de Dados e Processamento

O fluxo de dados segue uma sequência lógica, garantindo que os dados sejam processados, transformados e utilizados em cada etapa do ciclo de vida do ML:

1️⃣ Geração de Dados: O serviço API-DATA-SERVER-V1.PY (porta 8777) simula a geração contínua de dados de vendas de veículos.
2️⃣ Ingestão Orquestrada: O PREFECT-DATA-INJECTION-FLOW-V1.PY, orquestrado pelo Prefect, busca os dados da API de vendas e os envia para o Data Lake.
3️⃣ Armazenamento Bruto (Data Lake): A API-OBJECT-STORAGE-SERVER-V1.PY (porta 8778) recebe os dados e os armazena em um banco de dados SQLite (sales_data_lake.db), atuando como um "data lake" passivo.
4️⃣ Treinamento do Modelo ML: O ML-FLOW-DATA-PIPELINE-MODEL-TRAINER-GRAPH-REPORT-EXPORTER.PY lê os dados do Data Lake, pré-processa-os, treina um modelo Keras de previsão de vendas e registra o modelo (com seu pré-processador) no MLflow Model Registry (porta 5001). Relatórios e gráficos são gerados e enviados para o Artifact Storage (8779).
5️⃣ Serviço de Modelo: A API-SERVER-MODEL-LOAD-FROM-ML-FLOW--PREDICT-SERVICE-V1.PY (porta 8780) carrega a versão mais recente do modelo do MLflow Registry e expõe um endpoint para inferência (previsões).
6️⃣ Teste e Validação Contínua: O API-SERVER-MODEL-TESTER-V1.PY testa o endpoint de previsão (8780) com dados sintéticos, gerando relatórios de desempenho e validando a funcionalidade do modelo em produção.
7️⃣ Análise e Relatórios de Negócios: Paralelamente, o ML-FLOW-DATA-PIPELINE-GRAPH-GEN-V1.PY lê os dados do Data Lake, gera gráficos analíticos (ex: Top 10 modelos) e os armazena no MLflow e no Artifact Storage (8779).

🚀 Como Colocar o Pipeline em Funcionamento

Para iniciar e testar o pipeline completo, siga os passos na ordem indicada:

  1. Inicie o Servidor MLflow:
    Abra um terminal e execute: python ML-FLOW-SERVER-V1.PY
    Acesse a UI em: http://127.0.0.1:5001
  2. Inicie o Servidor Prefect:
    Abra outro terminal e execute: python PREFECT-SERVER-V1.PY
    Acesse a UI em: http://localhost:4200
  3. Inicie a API de Dados de Vendas (Fonte):
    Abra outro terminal e execute: python API-DATA-SERVER-V1.PY
    Esta API rodará na porta 8777.
  4. Inicie a API de Storage de Dados (Data Lake):
    Abra outro terminal e execute: python API-OBJECT-STORAGE-SERVER-V1.PY
    Esta API rodará na porta 8778.
  5. Inicie a API de Storage de Artefatos:
    Abra outro terminal e execute: python API-SERVER-DATA-STORAGE-OUTPUT-V1.PY
    Esta API rodará na porta 8779.
  6. Execute o Flow de Ingestão de Dados Prefect:
    Abra outro terminal e execute: python PREFECT-DATA-INJECTION-FLOW-V1.PY
    Isso buscará dados da porta 8777 e os enviará para a 8778. Você verá um "run" na UI do Prefect.
  7. Execute o Pipeline de Treinamento e Registro do Modelo MLflow:
    Abra outro terminal e execute: python ML-FLOW-DATA-PIPELINE-MODEL-TRAINER-GRAPH-REPORT-EXPORTER.PY
    Este pipeline buscará dados da 8778, treinará o modelo e o registrará no MLflow (5001), além de subir artefatos para 8779. Verifique a UI do MLflow para o "run" e o modelo registrado.
  8. Inicie a API de Serviço do Modelo (Inferência):
    Abra outro terminal e execute: python API-SERVER-MODEL-LOAD-FROM-ML-FLOW--PREDICT-SERVICE-V1.PY
    Esta API rodará na porta 8780 e carregará o modelo do MLflow.
  9. Execute o Tester da API de Serviço do Modelo:
    Abra outro terminal e execute: python API-SERVER-MODEL-TESTER-V1.PY
    Este script testará a API da porta 8780 e gerará relatórios de teste em test_data_reports/.
  10. Execute o Pipeline de Geração de Gráficos e Relatórios MLflow:
    Abra outro terminal e execute: python ML-FLOW-DATA-PIPELINE-GRAPH-GEN-V1.PY
    Este pipeline buscará dados da 8778, gerará gráficos e os registrará no MLflow (5001) e subirá para 8779.

Dica: Mantenha cada servidor (MLflow, Prefect, APIs Flask) rodando em terminais separados para facilitar o monitoramento e o Ctrl+C para encerrar.

🔮 Considerações e Próximos Passos

Este pipeline é uma base sólida para um ambiente MLOps. Para evoluções, podemos considerar:

  • 👀 Monitoramento Contínuo: Implementar monitoramento de deriva de dados e desempenho do modelo em produção.
  • 🔄 CI/CD para Modelos: Automatizar o deploy de novas versões de modelos e pipelines via integração e entrega contínuas.
  • 🔗 Orquestração End-to-End: Usar Prefect para orquestrar *todos* os pipelines (ingestão, treinamento, avaliação, deploy), não apenas a ingestão inicial.
  • 🚀 Auto-escalabilidade: Implementar soluções como Kubernetes para o deploy dos serviços e modelos.
  • 🧩 Feature Store: Introduzir uma Feature Store para gerenciar e servir features de forma consistente para treinamento e inferência.
